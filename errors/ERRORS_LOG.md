# ОШИБКИ REVA-SCANNER — Лог и решения

## КРИТИЧЕСКАЯ ОШИБКА #1 — JSON split bug
**Файл:** `app/services/full_scan_ai_service.py:217`
**Симптом:** `ValueError: Claude returned non-JSON for full scan report: ` (пустая строка)
**Причина:**
```python
# БЫЛО (баг):
raw_text.split("```", 2)[-1]
# Пример: "```json\n{...}\n```".split("```", 2) → ["", "json\n{...}\n", ""]
# [-1] даёт ПУСТУЮ СТРОКУ!

# ПРАВИЛЬНО:
raw_text.split("```", 2)[1]
# [1] даёт "json\n{...}\n" — нужный контент
```
**Статус:** ✅ ИСПРАВЛЕНО

## ОШИБКА #4 — Повторные вопросы (дата, имя)
**Файл:** `app/bot/handlers/full_scan.py`
**Симптом:** Бот каждый раз заново спрашивал дату рождения и имя

3. Автоматически сохраняет в `scan.answers` и пропускает вопрос
**Статус:** ✅ ИСПРАВЛЕНО

## ПОТЕНЦИАЛЬНАЯ ПРОБЛЕМА #7 — ai_service vs full_scan_ai_service
**Описание:** Два отдельных AI-сервиса с разными форматами вывода:
- `ai_service.py` → свободный текст (мини-скан, тизер)
- `full_scan_ai_service.py` → строгий JSON 6 блоков (полный скан)
**Риск:** Если Claude начнёт возвращать не-JSON для полного скана — ошибка
**Решение:** Добавлен более надёжный JSON-парсинг (см. исправление #1)
**Статус:** ⚠️ Под наблюдением — промпты требуют чёткой инструкции JSON

## ОШИБКА #8 — UnboundLocalError: InlineKeyboardMarkup
**Файл:** `app/bot/handlers/full_scan.py:178`
**Симптом:** `UnboundLocalError: local variable 'InlineKeyboardMarkup' referenced before assignment`
**Причина:**
```python
# В except-блоке (строка 135) был локальный импорт:
from aiogram.types import InlineKeyboardButton, InlineKeyboardMarkup
# Python помечал InlineKeyboardMarkup как локальную переменную ВЕЗДЕ в функции
# Строка 178 (вне except-блока) не могла её найти → UnboundLocalError
```
**Решение:** Удалён локальный импорт. Модульный импорт в начале файла достаточен.
**Статус:** ✅ ИСПРАВЛЕНО (01.04.2026)

## АРХИТЕКТУРНАЯ ЗАМЕТКА — Память пользователя
**Текущее состояние:**
- `users.birth_date` — сохраняется после первого ввода
- `users.full_name` — из Telegram профиля
- Ответы анкеты — в `scans.answers` (JSONB) — **не переиспользуются между сканами**

**Желаемое поведение (будущее):**
- Единый профиль пользователя накапливает данные между сканами
- Поле деятельности, роль, ситуация — не спрашиваются повторно
- История сканов влияет на глубину следующего разбора
