Create `app/bot/handlers/full_scan.py` with a single dynamic FSM engine (NOT 15 separate handlers). Architecture:

1. **Helper: `_send_question(bot, chat_id, question: QuestionDef, index: int, total: int)`**
   - Sends message with progress indicator: f"Вопрос {index + 1} из {total}\n\n{question.text}"
   - If input_type="keyboard": builds InlineKeyboardMarkup from question.options using prefix `fq:{key}:{value}` for callback_data
   - If input_type="text" and required=False: adds a "Пропустить" button with callback_data `fq_skip:{key}`
   - If input_type="text" and required=True: no skip button, just the text prompt
   - For birth_date question: add hint "Например: 15.05.1990"

2. **Helper: `_advance_to_next(state, session, scan_id, scan_type, current_index, bot, chat_id)`**
   - Calculates next_index = current_index + 1
   - If next_index >= total questions: call complete_questionnaire, set state to FullScanStates.completing, send "Анкета заполнена! Запускаю анализ..." message, then clear FSM state
   - If next_index < total: set FSM state to FullScanStates for next_index (use `getattr(FullScanStates, f"q{next_index}")`), call _send_question

3. **Entry point: `handle_buy_callback`** — handles callback_data "buy:personal" and "buy:business"
   - Extract scan_type from callback_data ("buy:personal" -> "personal")
   - Get or create user via UserService
   - Check for existing incomplete scan via get_incomplete_scan:
     - If incomplete scan exists with SAME scan_type: resume it (calculate current_index from answer count, send next question)
     - If incomplete scan exists with DIFFERENT scan_type: delete/cancel old scan, create new one
     - If no incomplete: create new full scan via create_full_scan
   - Store scan_id, user_id, scan_type, current_index in FSM data
   - Send first unanswered question via _send_question
   - Set FSM state to appropriate FullScanStates.qN

4. **Keyboard answer handler** — single handler for ALL keyboard questions:
   - Register with: `@router.callback_query(FullScanStates, lambda c: c.data.startswith("fq:"))`
   - Parse callback_data: `fq:{key}:{value}` -> extract key and value
   - Call scan_service.save_answer(scan_id, key, value)
   - Call _advance_to_next with current_index + 1
   - Answer the callback

5. **Skip handler** — for optional text questions:
   - Register with: `@router.callback_query(FullScanStates, lambda c: c.data.startswith("fq_skip:"))`
   - Parse key from callback_data
   - Save empty string as answer: save_answer(scan_id, key, "")
   - Call _advance_to_next

6. **Text input handler** — for text questions (birth_date, name, situation, social_url):
   - Register with: `@router.message(FullScanStates)` — catches ALL FullScanStates text
   - Get current question from FSM data (current_index + scan_type)
   - Validate:
     - If birth_date: use parse_birth_date from mini_scan module; on error reply with format hint and stay in state. On success, also call user_service.update_birth_date if not already set.
     - If max_length: truncate to max_length chars
     - If social_url: basic validation (starts with http:// or https:// or contains a dot), accept anything on failure but log warning
   - Save answer via save_answer
   - Call _advance_to_next

7. **FSM data structure** stored in state:
   ```python
   {
       "scan_id": int,
       "user_id": int,
       "scan_type": "personal" | "business",
       "current_index": int,  # 0-based question index
   }
   ```
